6/6日：工程师如何挑选创业公司

6月6日是台湾的工程师日，许多国家或地区设定了不同的工程师日或周，中国大陆暂时没有。 本文作者建议工程师在决定是否加入创业公司时，考虑如下几个问题：

  1. 公司是否以简单的思路解决了复杂的问题？
  2. 公司所在市场是不是够大？
  3. 公司CEO是否具备找钱、找客户和找人的能力？
  4. 团队的愿景和使命是不是你愿意投入的？与创始人见面两天后是否还希望与他一起工作？

要投入一项工作，就应该像投资人一样拷问创业公司。

6/5日：快乐生活的7个习惯

快乐源于内在的满足：亲密关系、合适的职业、内心的冲动... 扩展阅读：《寻找内在的自我 —— 马斯洛谈幸福》。

  1. 忙而不急，适度扩展舒适圈。
  2. 有5个亲密关系，可以深入交流重要问题，可以每两周联系一次。
  3. 不要依赖外在的评价。
  4. 享受运动。
  5. 掌握某项技能，其成就感远大于学习的痛苦。
  6. 花钱买经历而非物品。
  7. 不要忽视自己的渴望。

在论述第三点时，作者的一句引用让我琢磨了挺久：

Humility is not thinking less of yourself, but thinking of yourself less. — C.S. Lewis

我试着翻译如下：

谦卑不是贬低你自己，而是不要总想着你自己。— 英国作家C.S. Lewis

如果你有更好的理解，请让我知道。

6/4日：个人成长框架

本文作者发现最好的成长框架有利于“自我发现”和“专注力”。

  1. 动机，思考是什么驱使你做你所做的事？可与主管分享，每年更新一次。
  2. 价值观，是你做人生决策的依据，思考你的价值观是什么？可与主管分享，每年更新一次。
  3. 建立新的起点，思考你的优势和劣势是什么？每年更新一次。
  4. 设定成长目标，利用OKR（目标与关键结果），写下你想要达成的3个目标和每个目标的1到2个关键结果。安排相关日程，每周回顾一次，每季度更新一次。
  5. 了解工作中哪些方面给你带来活力，多做此类工作。做相关的心理测试，同步这些自我认知，每个季度一次。
  6. 成长需要专注力，养成一些积极主动的生活习惯，例如冥想或关掉手机。

6/3日：用SLC代替MVP

MVP（Minimal Viable Product）：最简化可行产品，由《精益创业》作者Eric Ries提出。创业公司使用MVP意味着低成本试错、获取市场反馈、快速迭代。本文作者认为用户讨厌MVP，他提出了SLC原则。

  1. 简单的（Simple），从功能到使用门槛。
  2. 完整的（Complete），产品必须提供完整的服务。
  3. 可爱的（Lovable），从界面设计到业务功能，体现的是公司态度和文化。

例如滑板就是一个SLC产品，不升级照样有很多人喜欢，但也可以加装扶手升级成滑板车，甚至继续改造成为汽车。

我觉得，作者是在描述什么是一个好的MVP，下次构建创建MVP时，不妨考虑SLC原则。

6/2日：如何建立文档文化

我一直认为，对知识工作者，写文档是一项高杠杆率的工作，我曾将文档产出作为绩效指标之一。本文作者提供了三个步骤，将文档融入公司文化。

  1. 从标准化模板开始：既简化了部分重复工作，又让分享者和使用者对文档有一致的预期。
  2. 建立活的文档：随着组织的成长和知识的更新，文档应该得到及时维护，以确保可用，减少重复工作。
  3. 授权所有成员：使用文档管理系统，鼓励所有成员做贡献，帮助公司成长。

6/1日：用Markdown写文档

我用Markdown写的文本，转换成HTML后，可直接复制到公众号，效率还不错。Markdown语法简单，我几乎只需要使用标题、列表、引用、超链接、图片等标记就可以完成排版。Markdown的高级或扩展功能还可以支持表格、代码、流程图和数学公式。

John Gruber开发了Markdown，他的独立博客Daring Fireball，专门评论苹果公司的产品和策略。从网站上可以看到，Markdown最早在2004年发布：

Markdown is a text-to-HTML conversion tool for web writers.

许多写作工具都支持Markdown语法，微信公众号还不支持。
